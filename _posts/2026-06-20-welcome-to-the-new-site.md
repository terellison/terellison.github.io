I finally gave this site the overhaul it deserved. It's now a proper home for
two things: the projects I build, and the things I learn while building them.

<!--more-->

## Why bother

For a long time this domain was a near-empty Jekyll theme with no posts and a
single "about" line. That's fine as a placeholder, but I wanted somewhere to
actually *show* work and *write* about it — without fighting the tooling every
time I have an idea.

So the goals were simple:

- A clean, fast, **portfolio** that pulls real projects front and center.
- A **blog** that's genuinely easy to publish to.
- Light and dark themes that respect your system preference.

## What you'll find here

The [Projects](/projects/) page collects the things I'm proud of — from
backend .NET tools like **NVUpdateManager** and a type-safe **RegistryWrapper**
library, to systems and graphics experiments in C++.

The blog is where I'll write about:

1. **.NET and C#** — patterns, gotchas, and the occasional deep dive.
2. **Systems programming** — the low-level stuff I tinker with for fun.
3. **Whatever I'm currently learning.** Writing it down is how it sticks.

## Publishing is a one-liner

The whole point of the redesign was to remove friction. Creating a new post is
now a single command:

```bash
bundle exec jekyll post "My Next Post"
```

That scaffolds a dated Markdown file with the right front matter, and I just
start writing. More on that workflow in the repo's `AUTHORING.md`.

Thanks for stopping by — there's more on the way.
